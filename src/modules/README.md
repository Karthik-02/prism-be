# Module Layout

- `auth`: implemented in this phase (`request-otp`, `verify-otp`, `logout`).
- `rbac`: permission resolution + middleware.
- `profile`: reserved for profile APIs.
- `users`: reserved for user lifecycle APIs.
- `roles`: reserved for role management APIs.
- `email-domains`: reserved for domain allowlist APIs.
- `prs`: reserved for PR lifecycle APIs.
- `releases`: reserved for release orchestration APIs.
- `release-notes`: reserved for release note APIs.
- `audit`: reserved for audit logging services.
- `notifications`: reserved for notification delivery/logging services.
